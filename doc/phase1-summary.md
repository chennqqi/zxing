## 技术架构

### 整体架构
```
Go应用层
    ↓
统一接口层 (pkg/zxing)
    ↓
后端实现层
├── CGO后端 (pkg/zxing/cgo_impl.go)
└── WASM后端 (pkg/zxing/wasm_impl.go)
    ↓
底层库
├── ZXingCPP C++库 (v2.3.0)
└── WebAssembly模块
```

### 核心组件
1. **ZXingCPP核心库**: 提供条码识别和解码的核心功能
2. **C++包装器**: 将ZXingCPP功能包装为C接口
3. **Go CGO接口**: 通过CGO调用C++库
4. **Go WASM接口**: 通过WASM调用C++库
5. **统一API**: 提供一致的Go语言接口

## 功能特性

### 支持的条码格式
- QR Code
- Code 128, Code 39, Code 93
- EAN-8, EAN-13
- UPC-A, UPC-E
- Data Matrix
- PDF417
- Aztec
- Codabar
- ITF
- MaxiCode

### 核心功能
- **解码功能**: 支持图像文件解码、字节数据解码
- **编码功能**: 支持文本编码为各种条码格式
- **批量处理**: 支持一次解码多个条码
- **配置选项**: 可配置的解码参数（旋转、反转、缩放等）

## 性能特点

### CGO方式
- **优势**: 性能好，与C++库直接集成
- **适用场景**: 服务器端应用、高性能要求场景

### WASM方式
- **优势**: 跨平台兼容性好，无需CGO环境
- **适用场景**: 浏览器环境、跨平台应用

## 构建和部署

### 构建要求
- **CGO构建**: Go 1.18+, C++17编译器, CMake 3.10+
- **WASM构建**: Emscripten SDK, CMake 3.10+

### 构建输出
- **CGO**: 静态库(.lib/.a) + 动态库(.dll/.so)
- **WASM**: .wasm文件 + .js加载器
